How it works

This site is connected to the same backend used by the enforcement system. Search results and payment status are read directly from the ticket database.

Shared record source The portal and the security dashboard use the same ticket data, so updates appear in both places.
Payment verification Payments are initialized through Paystack and verified on the server before re-entry restrictions are cleared.
Need support? If the ticket details look wrong, use the admin office or security control room process rather than forcing payment from the portal.
