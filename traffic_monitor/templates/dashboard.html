How operators should use this

Use this surface for visibility and exception handling, not as a public payment page. Tickets here are the internal enforcement records that drive entry restrictions.

Ticket review Inspect the ticket ID, recorded speed, and current status before changing any record manually.
Manual settlement Use “Mark Paid” only for verified offline or administrative settlements.
Data parity The admin dashboard and public portal read the same underlying ticket table, so status changes propagate immediately.

System notes

Field nodes should post event payloads to /api/v1/log_event. The gate node should keep using /api/v1/check_ticket?plate=... for enforcement checks.
